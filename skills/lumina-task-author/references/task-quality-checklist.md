# Task Quality Checklist

- Goal is user-centric and measurable.
- Scope is narrow enough for one PR.
- Out-of-scope prevents feature creep.
- File list includes only intended touch points.
- DoD includes:
  - build passes
  - no hardcoded user strings
  - i18n keys present in ka/en/ru
  - manual verification notes
- Do NOT section includes schema/auth guardrails when relevant.
